# 컬렉션의 정의

컬렉션은 특별한 목적을 가진 파일과 사람의 집합입니다. 예를 들어 다음과 같은 목적을 가지고 컬렉션을 만들 수 있습니다.

- 업무 관리
- 외주 업체 관리
- 신규 입사자를 위한 온보딩 페이지 제작
- 나만의 메모장
- …

이러한 다양한 목적을 달성하는걸 돕기 위해 컬렉션은 아래와 같은 특별한 기능을 제공합니다.

- 컬렉션 내에서만 적용되는 파일 속성
- 외부 링크 바로가기 추가
- 외부인이 파일 조회 및 업로드가 가능한 링크 생성
- 파일 추가 요청
- 칸반 보기 기능

단순한 기능이지만 이러한 기능으로 정말 많은 업무상의 목적을 반영한 특별한 파일 관리 페이지를 만들 수 있습니다. 무엇보다, AI가 자연스럽게 업무의 목적을 이해할 수 있습니다. 업무의 목적을 이해한 AI는 그 어떤 AI보다 만족스러운 결과를 제공합니다. (오래 일한 동료와 합이 잘 맞는 것과 비슷합니다!)

컬렉션을 잘 활용하여 팀원과 AI의 성과를 120% 끌어올려보세요!

> 파일을 컬렉션에 저장되는 개념이 아닙니다. 컬렉션으로 지정된 파일을 모아서 보는 개념이기 때문에 한 파일은 여러 컬렉션에 속할 수 있습니다.

# 컬렉션의 UI

## 1. 보기 영역

컬렉션에 포함되어 있는 다양한 파일이 노출되는 영역입니다. 보기 영역에 대한 설정은 [2. 필터 영역]에서 이루어집니다. 전체 파일 페이지와 비슷하지만 아래와 같은 차이가 있습니다.

- 컬렉션에 참여하고 있는 모든 사람들이 같은 필터를 공유한다.
- 칸반 보기를 지원한다.
- 외부 링크 바로가기와 파일 추가 요청이 가능하다. (추후 지원 예정)

파일을 클릭 시 파일이 선택되며 아래와 같은 메뉴가 보여집니다. 더블 클릭 시 해당 파일 상세 페이지로 이동합니다.

## 2. 필터 영역

- **검색**
    
    컬렉션 내에 있는 파일, 바로가기, 요청을 검색을 할 수 있습니다. 사이드바의 검색 기능과 동일한 AI 검색이지만, 검색 결과 페이지로 이동하지 않으며, 컬렉션 페이지 내에서 보기 영역에 필터링이 되는 방식입니다. 
    
- **필터**
    
    파일, 바로가기, 요청에 지정된 담당자, 진행 상황 등의 속성에 필터를 설정할 수 있습니다.
    
- **그룹**
    
    보기 영역에서 파일들을 분류하는 기준 속성을 설정합니다. 진행 상황을 그룹 기준으로 설정할 경우 진행 상황의 유형인 “None”, “Needs Edit”, “In Progress”, “Done”으로 파일이 그룹지어 집니다.
    
- **설정**
    
    각 파일 컴포넌트에 노출할 속성을 설정합니다. 칸반뷰에서는 기본적으로 파일 썸네일, 파일명만 노출됩니다.
    
- **보기 종류 선택 드롭다운**
    
    보기 영역의 종류를 설정합니다. 갤러리 보기, 리스트 보기, 칸반 보기 중 한가지를 선택할 수 있습니다.
